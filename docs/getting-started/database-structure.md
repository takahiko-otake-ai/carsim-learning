# データベース構造

CarSim のデータ管理は **Program フォルダ**と **Database フォルダ**の 2 層構造で構成されています。この構造を理解することで、効率的なデータ管理とチームでの共有が可能になります。

## Program フォルダと Database フォルダ

### Program フォルダ

PC に 1 つ存在し、通常は `C:\Program Files\CarSim_XX.X\` にインストールされます。以下の要素が含まれます。

- **ソルバー** — 車両運動の計算を行う実行ファイル
- **GUI（VS Browser）** — パラメータ設定用のインターフェース
- **VS Visualizer** — 結果の 3D アニメーション表示ツール
- **アニメーションアセット** — 車両・道路・環境の 3D モデル
- **ヘルプドキュメント** — リファレンスマニュアル
- **サンプルアーカイブ（.cpar）** — Database Builder で展開されるサンプルデータ

### Database フォルダ

PC に複数作成でき、プロジェクトごとにデータを分離して管理します。以下の要素が含まれます。

- **Parsfiles（パラメータファイル）** — シミュレーション設定ファイル
- **Simfiles** — ソルバーへの入力ファイル
- **Simulink モデル** — MATLAB/Simulink 連携用モデル
- **道路ファイル** — 道路形状の定義データ
- **結果ファイル** — シミュレーション出力データ

:::info
Database フォルダはユーザーが自由に作成・削除できるため、プロジェクトや実験条件ごとにデータベースを分けて管理することを推奨します。
:::

## ライブラリの概念

CarSim では、シミュレーションの各設定項目を**ライブラリ**として分類・管理しています。ライブラリはシミュレーションの構成要素であり、Database フォルダ内のサブフォルダとして存在します。

主要なライブラリの例：

| ライブラリ | 役割 |
|------------|------|
| **Run Control** | シミュレーション全体の実行設定 |
| **Vehicle Assembly** | 車両モデルの構成 |
| **Procedure** | 走行手順（操舵・加減速の入力） |
| **Road** | 道路環境の定義 |
| **Camera Setup** | VS Visualizer のカメラ設定 |

## データセットと Parsfile

各ライブラリには複数の**データセット**を登録できます。データセットの実体は `.par` 拡張子の**パラメータファイル（Parsfile）**です。

- 1 つのデータセットが 1 つの Parsfile に対応します
- データセットに設定されたパラメータがシミュレーションの挙動を決定します
- データセットの命名規則: `[Database] Library {Category} Title` の形式

:::tip
命名規則に従ってデータセットを整理すると、ライブラリ内でのデータセット検索が容易になります。カテゴリを活用して論理的にグループ分けしましょう。
:::

## エクスポートとインポート

CarSim では 2 つのファイル形式でデータの受け渡しができます。

- **Consolidated Parsfile（.cpar）** — 関連するすべてのデータセットを 1 つのアーカイブファイルにまとめた形式です。データベース間の移動やバックアップに使用します。
- **Expanded Parsfile（.par）** — 個別のパラメータファイルです。テキスト形式のため、内容の確認や手動編集が可能です。

## Library Tool

**Library Tool** を使うと、データセットに対するバッチ操作が可能です。複数のデータセットをまとめてエクスポート・削除・カテゴリ変更するなど、大量のデータを効率的に管理できます。

## External Database（ver 2025.0 以降）

バージョン 2025.0 から導入された **External Database** 機能により、データベースをバージョン管理システムと連携させたり、チーム内でデータを配布したりすることが容易になりました。

- Git などのバージョン管理ツールとの統合が可能
- チーム間でのデータセット共有・配布を効率化
- データの変更履歴を追跡可能

:::note
External Database は比較的新しい機能です。チーム開発やデータ管理の効率化に大きく貢献するため、積極的に活用を検討してください。
:::
