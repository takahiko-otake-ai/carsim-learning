# GUI の基本操作

CarSim の GUI（**VS Browser**）は、シミュレーションの設定・実行・結果確認を統合的に行うためのインターフェースです。ここでは主要なツールバーボタンとデータセット管理の基本を解説します。

## ツールバーの概要

VS Browser の上部にはツールバーが配置されており、頻繁に使う機能にすばやくアクセスできます。

### ナビゲーション

- **Back / Forward（戻る・進む）** — Web ブラウザと同様に、画面の閲覧履歴を前後に移動します
- **Previous / Next Dataset（前・次のデータセット）** — 同じライブラリ内でデータセットを順に切り替えます

### 主要なツールバーボタン

| ボタン | 機能 |
|--------|------|
| **Duplicate** | 現在のデータセットの複製を作成 |
| **Copy** | データセットのコピーとリンク |
| **Run Control** | ホーム画面（Run Control）に戻る |
| **Library Browser** | ライブラリ一覧を開いてデータセットを検索・選択 |
| **Parsfile Open** | パラメータファイルを直接開く |
| **Dataset Delete** | 現在のデータセットを削除 |
| **Sidebar** | サイドパネルの表示/非表示を切り替え |
| **Help** | ヘルプドキュメントを表示 |
| **Update** | データベースの更新を確認 |
| **Unlock / Lock** | データセットの編集ロック/ロック解除 |

## データセット管理：Duplicate と Copy and Link

データセットの複製には 2 つの方法があり、それぞれ挙動が異なります。

### Duplicate（複製）

現在のデータセットの独立したコピーを作成します。元のデータセットとの関連は持たず、完全に独立した新しいデータセットになります。自由に変更を加えたい場合に使用します。

### Copy and Link（コピーとリンク）

データセットのコピーを作成すると同時に、そのコピーを現在参照中のデータセットのリンク先として設定します。上位のデータセットから参照されるサブデータセットを複製して差し替えたい場合に便利です。

:::warning
Duplicate と Copy and Link の違いを理解しておくことは重要です。意図しないリンクの書き換えを防ぐため、目的に応じて使い分けてください。
:::

## ファイルの確認

VS Browser からシミュレーションに関連する各種ファイルを確認できます。

- **Echo file** — シミュレーション実行時に読み込まれた全パラメータの記録
- **Log file** — 実行ログ（警告やエラー情報を含む）
- **Simulation results** — シミュレーション結果ファイル
- **VS/ERD header** — 結果ファイルのヘッダー情報（出力変数の定義）
- **State variables** — 状態変数の一覧
- **Imports / Outports** — 外部連携用の入出力ポート定義

:::tip
シミュレーション結果に問題がある場合は、まず **Log file** を確認してエラーや警告の有無をチェックしましょう。
:::

## シミュレーション実行の流れ

CarSim でシミュレーションを実行すると、内部では以下の処理が順に行われます。

1. **Simfile の作成** — GUI の設定内容からシミュレーション入力ファイル（Simfile）が生成されます
2. **Run_all.par の生成** — 実行に必要な全パラメータを統合した制御ファイルが作成されます
3. **ソルバーによる計算** — CarSim ソルバーが Simfile を読み込み、車両運動の計算を実行します
4. **結果ファイルの生成** — 計算結果が `.vs` 形式などのファイルとして出力されます

この流れを理解しておくと、トラブルシューティングや高度なカスタマイズの際に役立ちます。
