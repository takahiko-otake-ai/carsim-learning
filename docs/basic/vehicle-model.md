# 車両モデルの概要

CarSimは、常微分方程式による因果モデリングに基づく**4輪フルビークルモデル**を採用しています。個々のコンポーネント機構を直接モデル化するのではなく、特性値入力データ（トー変化、キャンバー変化、タイヤ特性、ステアリング機構、エンジン特性、トルクコンバータ、ブレーキなど）を用いて車両挙動を再現します。

:::info 計算性能
標準的なWindows PCにおいて、基本シミュレーションは**リアルタイムの15倍以上**の速度で実行可能です。振動・乗り心地シミュレーションは約20Hzまで対応します。
:::

## Vehicle Assembly画面

Vehicle Assembly画面は、CarSimにおける車両モデルの中心的な設定画面です。ここからすべての主要サブシステムにアクセスできます。

| リンク先 | 設定内容 |
|---|---|
| **Sprung Mass** | ばね上質量（車体）の慣性特性 |
| **Aerodynamics** | 空力係数・風の設定 |
| **Powertrain** | エンジン、モーター、変速機、デフ |
| **Brakes** | ブレーキシステムの特性 |
| **Steering** | ステアリング機構の設定 |
| **Suspension** | サスペンションのキネマティクスとコンプライアンス |
| **Tires** | タイヤモデルの選択と設定 |

## 座標系

CarSimは**右手座標系**を採用しています。

- **X軸**: 車両前方が正
- **Y軸**: 車両左方が正
- **Z軸**: 上方が正
- **原点**: フロントアクスル中心、おおよそ地面レベル

:::note 座標系の原点
原点がフロントアクスル中心に設定されている点は、各種パラメータの入力時に特に注意が必要です。ホイールベースや重心位置などはこの原点を基準として定義されます。
:::

## ばね上質量（Sprung Mass）

ばね上質量は車体を表す剛体モデルです。以下の慣性特性を定義します。

- **質量**: 車体の総質量
- **重心位置**: 原点（フロントアクスル中心）からの相対座標（X, Y, Z）
- **慣性モーメント**: ロール、ピッチ、ヨー軸まわりの慣性モーメントおよび慣性乗積

## 初期化プロセス

シミュレーション開始時、CarSimは**1G平衡状態の計算**を自動的に実施します。この初期化プロセスでは以下が行われます。

1. 車両が静止した状態で、重力による荷重配分を計算
2. 各サスペンションの初期変位を求解
3. タイヤの接地荷重を算出
4. すべてのサブシステムが整合する平衡状態を決定

:::tip Vehicle Code
Vehicle Codeを活用すると、特定の車両設定を一意に識別・管理できます。異なるシミュレーション間で同じ車両パラメータセットを再利用する場合や、複数の車両バリエーションを管理する場合に便利です。
:::

## モデリングの特徴

CarSimのモデリングアプローチには以下の特徴があります。

- **コンプライアンス入力**: ステアリングやサスペンションではコンプライアンス（弾性変形）を考慮
- **特性値ベース**: 部品の物理的な形状・機構ではなく、計測・推定された特性値を入力として使用
- **高速演算**: 常微分方程式による定式化により、高い計算効率を実現
- **周波数範囲**: 約20Hzまでの車両挙動を再現可能（高周波振動は対象外）
