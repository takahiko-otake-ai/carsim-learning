# 交通シナリオ

CarSimでは、道路・ムービングオブジェクト・ADASセンサ・イベントを組み合わせて、実際の交通状況を再現するシナリオを構築できます。

## シナリオ構築ワークフロー

交通シナリオは以下の手順で構築します。

```
道路を定義 → ムービングオブジェクトを配置 → センサを設定
    → イベントで条件・トリガーを設定 → シミュレーション実行
```

### 1. 道路の定義

走行環境の基盤となるパスと道路を設定します。Segment BuilderやVS Scene Builderを活用して、テストに必要な道路形状を構築します。

### 2. ムービングオブジェクトの配置

周囲車両や歩行者をパス上に配置し、速度・加速度プロファイルを設定します。先行車両、対向車両、横断歩行者など、シナリオに応じたオブジェクトを追加します。

### 3. センサの設定

自車両にADASセンサを搭載し、検知範囲や感度を設定します。テスト対象のADASシステムに合わせたセンサ構成を定義します。

### 4. イベントの設定

トリガー条件と、条件成立時のアクションを設定します。距離、時間、速度などの条件で、オブジェクトの挙動変化やシミュレーション終了を制御します。

### 5. シミュレーション実行

構築したシナリオを実行し、結果をVS Visualizerで確認します。

## 代表的なシナリオ例

### ACC（アダプティブクルーズコントロール）テスト

先行車両を配置し、車間距離の追従制御を検証します。先行車の加減速に対する自車両の応答を評価できます。

### 車線変更シナリオ

複数の周囲車両を配置し、車線変更時の安全性を検証します。隣接車線の車両との相対位置・速度を監視しながら、ADAS制御の適切性を評価します。

### 緊急ブレーキシナリオ

先行車両の急停止や歩行者の飛び出しを再現し、AEB（自動緊急ブレーキ）の動作を検証します。

### 交差点シナリオ

交差点での右左折時における対向車・歩行者との干渉を検証します。VS Scene Builderで構築した交差点環境を活用します。

:::tip
シナリオは段階的に複雑さを増やしていくことを推奨します。まず単純なシナリオで基本動作を確認し、その後にオブジェクト数や条件分岐を追加していくと、デバッグが容易になります。
:::

## 自動テストとの連携

### パラメータスタディ

構築したシナリオをベースに、車両パラメータやシナリオ条件を系統的に変化させたバッチシミュレーションを実行できます。

### VS COM Interface

**VS COM Interface**を使用すると、外部プログラムからCarSimのシミュレーションをバッチ実行できます。

- シミュレーション条件のプログラム的な変更
- 大量のテストケースの自動実行
- 結果の自動収集と後処理

:::info
VS COM Interfaceは、MATLAB/Simulink、Python、C++など多様な言語・環境から利用可能です。CI/CDパイプラインに組み込むことで、車両パラメータ変更時の回帰テストを自動化できます。
:::
