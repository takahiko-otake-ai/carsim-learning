# プロシージャとイベント

プロシージャデータセットは、CarSimにおける走行シナリオの中核を定義します。イベント機能と組み合わせることで、条件分岐を含む複雑なシナリオを構築できます。

## イベント機能

イベントはCarSimのシナリオ作成機能です。シミュレーション実行中に**トリガー（条件）**が成立すると、Parsfileの読み込みが発生し、シミュレーションの終了やパラメータの変更が行われます。

### イベントで変更可能な項目

- ドライバー操作入力
- 車両仕様パラメータ
- 道路設定

### 複数分岐

複数のトリガー条件を設定し、状況に応じた分岐シナリオを構築できます。

:::note
イベントでは**ソルバーレベルの設定**を変更できません。以下の項目はシミュレーション開始前に確定させる必要があります。
- 時間ステップ
- 変数定義
- サスペンション構成の変更
:::

## シミュレーション実行フロー

CarSimのシミュレーションは以下の3段階で実行されます。

```
┌─────────────────────────────────────────────┐
│  1. 初期化                                    │
│     Simfile + Run_all.par 生成                │
│     パラメータ読み込み → 初期計算              │
├─────────────────────────────────────────────┤
│  2. 積分計算                                  │
│     ソルバーDLLによるステップ計算              │
│     イベントのトリガー監視 → Parsfile読み込み    │
├─────────────────────────────────────────────┤
│  3. 終了処理                                  │
│     結果ファイルの生成                         │
└─────────────────────────────────────────────┘
```

### 各段階の詳細

1. **初期化**: `Simfile` と `Run_all.par` が生成され、すべてのパラメータが読み込まれます。初期状態の計算が行われ、車両モデルやドライバーモデルが準備されます。
2. **積分計算**: ソルバーDLLがタイムステップごとに車両運動方程式を解きます。この段階でイベントのトリガー条件が監視され、条件成立時にはパラメータが動的に変更されます。
3. **終了処理**: シミュレーションが完了すると、結果ファイルが生成されます。

## 出力ファイル

シミュレーション終了後に生成されるファイルは以下の通りです。

| ファイル種別 | 内容 |
|------------|------|
| シミュレーション結果 | 時系列データ（各種物理量） |
| Echoファイル | 設定パラメータの記録 |
| Logファイル | 実行時のログ情報 |

## Plot Setup

シミュレーション結果の表示をカスタマイズする機能です。

- 出力ファイルからの**信号選択**
- 凡例・ラベル・色・線種の設定
- **新しい変数**のプロット追加

:::tip
Plot Setupを活用すると、複数のシミュレーション結果を重ね合わせて比較できます。繰り返し使用するプロット設定はテンプレートとして保存しておくと効率的です。
:::
