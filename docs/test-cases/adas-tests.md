# ADAS テスト

`ADAS` カテゴリには、先進運転支援システム（Advanced Driver Assistance Systems）の開発・検証に必要なシミュレーションシナリオが収録されています。CarSimは、センサーモデル、移動オブジェクト、イベントトリガーを組み合わせることで、複雑な ADAS シナリオを構築できます。

## ADAS シナリオの構成要素

### Moving Objects（移動オブジェクト）

シミュレーション空間内に**ターゲット車両**や**歩行者**を配置します。各オブジェクトには、走行経路（Path）、速度プロファイル、出現・消滅タイミングを個別に設定できます。

- **ターゲット車両** — 先行車、対向車、交差車両など
- **歩行者** — 横断歩行者、路上歩行者
- **二輪車** — 自転車、オートバイ

### ADAS Sensors（センサー）

自車両に搭載する検知センサーを定義します。センサーの種類と配置を適切に設定することで、実際のセンサーに近い検知挙動を再現できます。

### Events（イベント）

**トリガー条件**に基づいてシミュレーション中にアクションを実行します。例えば「TTC が 2 秒以下になったらブレーキを入力」のような条件付きロジックを構築できます。

## 代表的なテストシナリオ

### ACC（アダプティブクルーズコントロール）

先行車に追従する走行シナリオです。

- **シナリオ**：自車が設定速度で走行中、前方の先行車に接近し、車間距離を維持しながら追従
- **評価項目**：車間距離制御精度、加減速の滑らかさ、先行車離脱時の加速復帰
- **Moving Objects**：先行車（速度変動あり）

### AEB（自動緊急ブレーキ）

障害物を検知して自動制動を行うシナリオです。

- **シナリオ**：走行中の前方に静止車両または歩行者が出現、センサーで検知し自動ブレーキを作動
- **評価項目**：検知距離、ブレーキ作動タイミング、衝突回避の可否、残存速度
- **Moving Objects**：静止ターゲット車両、横断歩行者

### LKA（レーンキーピングアシスト）

車線逸脱を検知し、ステアリング補正を行うシナリオです。

- **シナリオ**：車線中央からの逸脱を検知し、ステアリングトルクで車線内に復帰
- **評価項目**：車線内偏差、修正操舵の応答性、ドライバーへの介入度合い
- **Road**：車線マーキング付きの道路定義が必要

### 車線変更アシスト（Lane Change Assist）

安全な車線変更を支援するシナリオです。隣接車線のターゲット車両を検知し、安全な車線変更可否を判断します。

## センサーの設定

ADAS センサーの設定では、以下のパラメータを調整します。

| パラメータ | 説明 | 典型的な値 |
|-----------|------|-----------|
| 搭載位置 | 車両上のXYZ座標 | フロントバンパー中央など |
| 検知範囲 | 最大検知距離 | 50–200 m |
| 視野角（FOV） | 水平・垂直の検知角度 | ±30° / ±10° |
| 感度 | 検知の閾値設定 | シナリオにより調整 |
| オクルージョン | 遮蔽物による検知遮断 | ON / OFF |

:::note センサーモデルの選択
CarSimには簡易的なセンサーモデルが含まれていますが、より高精度なセンサーシミュレーションが必要な場合は、Simulink や外部ツールとの連携により、詳細なセンサーモデルを統合できます。
:::

## オブジェクトの設定

ターゲットオブジェクトには以下の属性を設定します。

- **サイズ** — 全長、全幅、全高（車両タイプに応じた寸法）
- **走行経路（Path）** — XY座標で定義された移動軌跡
- **速度プロファイル** — 時間に対する速度変化パターン
- **外観** — 3Dアニメーション表示用のモデル選択

## Simulink との連携

ADAS 制御アルゴリズムの開発では、**Simulink** との連携が効果的です。

1. CarSim で車両ダイナミクスとシナリオ環境を提供
2. Simulink で制御ロジック（ACC、AEB、LKA）を実装
3. 共同シミュレーションで統合テストを実施

この構成により、制御アルゴリズムのラピッドプロトタイピングとパラメータチューニングが可能になります。

:::tip
Simulink 連携の詳細については、[Simulink 連携](/docs/integration/simulink)のセクションを参照してください。
:::

## Euro NCAP テストシナリオ

Euro NCAP で定義されている ADAS 評価シナリオにも対応しています。

| シナリオカテゴリ | 内容 | 条件 |
|----------------|------|------|
| **Car-to-Car（CCR）** | 先行車への追突回避 | 静止車、減速車、低速車 |
| **Car-to-Pedestrian（CPR）** | 歩行者との衝突回避 | 成人横断、子供飛び出し |
| **Car-to-Cyclist（CCyR）** | 自転車との衝突回避 | 縦断走行、横断 |

各シナリオには、Euro NCAP プロトコルに準じた速度条件、オブジェクト配置、評価基準が設定されています。

## ADAS 評価の出力信号

ADAS テストでは、以下の出力信号を確認して制御性能を評価します。

- **TTC（Time to Collision）** — 衝突までの予測時間
- **相対距離（Relative Distance）** — 自車とターゲット間の距離
- **相対速度（Relative Speed）** — 自車とターゲット間の速度差
- **ブレーキ作動タイミング** — 制御介入の開始時刻
- **横偏差（Lateral Offset）** — 車線中央からの偏差量（LKA 評価）

:::info
ADAS テストでは、シナリオの再現性が重要です。同一シナリオで制御パラメータのみを変更し、その効果を定量的に比較する方法が推奨されます。
:::
