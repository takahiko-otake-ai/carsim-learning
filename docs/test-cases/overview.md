# テストケースの概要

CarSimには、車両ダイナミクスシミュレーションをすぐに始められるように、**500以上のサンプルデータセット**が付属しています。これらのデータセットは、目的別にカテゴリ分けされており、CarSimの機能を効率的に学習できるよう設計されています。

## データベースの構成

CarSimのサンプルデータベースは、以下の主要カテゴリで構成されています。

| カテゴリ | フォルダ名 | 内容 |
|---------|-----------|------|
| Quick Start ガイド | `Quick_Start` | 基本操作を学ぶ入門向けテストケース |
| 操縦安定性テスト | `Handling` | ステア応答、DLC、定常円旋回など |
| ブレーキテスト | `Brake` | 直進制動、ABS評価、旋回中制動など |
| ADAS テスト | `ADAS` | ACC、AEB、LKA などの先進運転支援 |
| EV/HEV テスト | `EV` | 電動パワートレイン、回生制動、SOC管理 |
| Simulink 連携 | `Simulink` | MATLAB/Simulink との共同シミュレーション |
| Embedded Python | `Embedded_Python` | Python スクリプトによる拡張機能 |

## カテゴリとタイトルによるデータ管理

CarSimのすべてのデータセットは**カテゴリ（Category）**と**タイトル（Title）**の2階層で整理されています。カテゴリはデータの大分類を表し、タイトルは個々のデータセットを識別する名前です。この構造により、数百のデータセットの中から目的のテストケースを素早く見つけることができます。

## Run Control からのナビゲーション

テストケースを探すには、**Run Control** 画面の **Datasets** タブを使用します。

1. Run Control 画面を開く
2. **Datasets** タブをクリック
3. カテゴリドロップダウンで目的のカテゴリを選択
4. 表示されるデータセット一覧から目的のテストケースを選択

:::tip ヒント
Datasets タブの検索機能を使えば、キーワードでデータセットをフィルタリングできます。例えば「DLC」と入力すれば、Double Lane Change に関連するすべてのテストケースが表示されます。
:::

## テストケースの構成要素

各テストケースは、個別のコンポーネントデータセットへのリンクで構成されています。

- **Vehicle Assembly（車両アセンブリ）** — 車体、サスペンション、タイヤなどをまとめた車両全体の定義
- **Procedure（手順）** — ドライバー入力、速度制御、操舵パターンなどの走行シナリオ
- **Road（道路）** — 路面形状、摩擦係数、レーンマーキングなどの道路定義
- **その他のコンポーネント** — ブレーキシステム、パワートレイン、ADAS センサーなど

この分離された構造により、同じ車両で異なる手順を実行したり、同じ手順で異なる車両を比較したりすることが容易になります。

## カスタムテストスイートの構築

CarSimの付属サンプルをベースに、独自のテストスイートを構築できます。

1. 既存のサンプルデータセットを**コピー**する
2. 必要なパラメータ（速度、操舵、路面条件など）を変更する
3. カテゴリとタイトルを自社の命名規則に合わせて設定する
4. 関連するテストケースを同一カテゴリにまとめる

:::info
サンプルデータセットは直接編集せず、必ずコピーを作成してからカスタマイズしてください。CarSimのアップデート時にサンプルデータが上書きされる可能性があります。
:::

複数のテストケースを連続実行する場合は、**Run Math Model** の一括実行機能（Batch Run）を活用できます。これにより、パラメータスタディや車両間比較を効率的に行えます。
