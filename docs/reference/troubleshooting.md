# トラブルシューティング

CarSim を使用する際に発生しやすい問題と、その解決方法をカテゴリ別にまとめました。

---

## 起動時の問題

### License not found（ライセンスが見つからない）

**症状:** CarSim 起動時に「License not found」エラーが表示される。

**解決方法:**

1. **ドングルライセンスの場合:** USB ドングルが正しく接続されているか確認してください。別の USB ポートに差し替えて試してください
2. **ネットワークライセンスの場合:** ライセンスサーバーへのネットワーク接続を確認してください。ファイアウォールがポートをブロックしていないか確認してください
3. CarSim GUI の **License Settings** 画面で、ライセンスの種類と設定が正しいか確認してください

:::tip
ライセンスの問題が解決しない場合は、Mechanical Simulation 社のサポートに連絡してください。
:::

### Database won't open（データベースが開かない）

**症状:** CarSim 起動時にデータベースの読み込みに失敗する。

**解決方法:**

1. データベースフォルダのパスに**日本語や特殊文字が含まれていないか**確認してください。パスには英数字のみを使用してください
2. データベースフォルダが存在し、アクセス権限があることを確認してください
3. データベースフォルダ内の必要なファイルが揃っているか確認してください

## シミュレーション実行時の問題

### Solver crash（ソルバーがクラッシュする）

**症状:** シミュレーション開始直後または途中でソルバーが異常終了する。

**解決方法:**

1. **Echo File を確認:** Results フォルダ内の Echo File を開き、エラーメッセージを確認してください。パラメータの読み込みエラーが記録されている場合があります
2. **Vehicle Code を確認:** 使用している Vehicle Code がライセンスで許可されているか確認してください
3. **パラメータ値を確認:** 物理的に不合理な値（負の質量、ゼロのタイヤ半径など）が設定されていないか確認してください

:::danger
Echo File にエラーが表示されている場合、該当するパラメータを修正してから再度実行してください。
:::

### Simulation runs but results look wrong（結果が不正確に見える）

**症状:** シミュレーションは完了するが、結果が予想と大きく異なる。

**解決方法:**

1. **単位を確認:** 入力パラメータの単位が正しいか確認してください（例: mm と m の混同、deg と rad の混同）
2. **入力データの範囲:** パラメータ値が現実的な範囲内かどうか確認してください
3. **Echo File の警告メッセージ:** エラーではなく警告（Warning）が出力されていないか確認してください
4. **基本モデルからの確認:** まず標準ライブラリのデフォルト車両で同じシナリオを実行し、自分のパラメータとの差異を比較してください

### Simulation very slow（シミュレーションが非常に遅い）

**症状:** シミュレーションの実行速度がリアルタイムよりも大幅に遅い。

**解決方法:**

1. **タイムステップ:** 必要以上に小さいタイムステップを設定していないか確認してください（推奨: 0.0005 秒）
2. **出力変数:** 不要な出力変数を減らしてください。大量の変数を出力するとファイル書き込みに時間がかかります
3. **外部モデル接続:** Simulink や他の外部モデルとの接続がボトルネックになっていないか確認してください

## 結果表示の問題

### VS Visualizer won't open（VS Visualizer が起動しない）

**症状:** シミュレーション後に VS Visualizer が起動しない、またはエラーが表示される。

**解決方法:**

1. **結果ファイル:** Results フォルダにシミュレーション結果ファイル（.vs, .erd など）が存在するか確認してください
2. **VS Visualizer のパス:** Run Control 画面で VS Visualizer の実行ファイルパスが正しく設定されているか確認してください

### Plots show no data（グラフにデータが表示されない）

**症状:** Plot 画面を開いてもデータが表示されない。

**解決方法:**

1. **出力変数の選択:** 表示したい変数が **Plot Setup** で出力変数として選択されているか確認してください
2. **シミュレーションの再実行:** 出力変数を追加した場合は、シミュレーションを再度実行する必要があります

### Animation looks incorrect（アニメーション表示が不正確）

**症状:** 3D アニメーションで車両形状がおかしい、またはオブジェクトが表示されない。

**解決方法:**

1. **アニメーションアセット:** 車両や道路の 3D アセットファイルのパスが正しいか確認してください
2. **アセットの存在:** 参照先のアセットファイルが実際に存在するか確認してください

## Simulink 連携の問題

### S-Function not found（S-Function が見つからない）

**症状:** Simulink モデル実行時に CarSim の S-Function が見つからないエラーが出る。

**解決方法:**

1. **MATLAB パスの追加:** CarSim の MATLAB 関連フォルダを MATLAB のパスに追加してください
2. **必要ファイルのコピー:** S-Function の MEX ファイルを Simulink モデルと同じフォルダにコピーしてください

:::note
CarSim のバージョンによって必要な MEX ファイルが異なります。対応する MATLAB バージョンも確認してください。
:::

### Simfile error（Simfile エラー）

**症状:** Simulink 実行時に Simfile の読み込みエラーが発生する。

**解決方法:**

1. CarSim GUI で **Generate Files for this Run** ボタンを押して、必要なファイルを生成してください
2. Simulink モデルの **Working Directory** が正しく設定されているか確認してください
3. Simfile のパスが Simulink ブロックに正しく指定されているか確認してください

### I/O port mismatch（I/O ポートの不一致）

**症状:** Simulink モデルのポート数が CarSim の設定と一致しないエラーが出る。

**解決方法:**

1. CarSim GUI で **Import/Export Channels** の設定を確認してください
2. Simulink モデルの S-Function ブロックの入出力ポート数が、CarSim で設定した Import/Export チャネル数と一致しているか確認してください
3. チャネル設定を変更した場合は、**Generate Files for this Run** を再度実行してください

## データ管理の問題

### Import fails（インポートに失敗する）

**症状:** .cpar ファイルのインポート時にエラーが発生する。

**解決方法:**

1. **バージョン互換性:** cpar ファイルが現在の CarSim バージョンと互換性があるか確認してください。古いバージョンで作成されたファイルは読み込めない場合があります
2. **UUID 競合:** 既存のデータベースに同じ UUID のデータセットが存在すると競合が発生します。インポート時のオプションで上書きまたはスキップを選択してください

### Cannot edit dataset（データセットを編集できない）

**症状:** データセットのパラメータを変更しようとしても入力欄がグレーアウトしている。

**解決方法:**

1. データセットの **Lock** 状態を確認してください
2. ロックされている場合は、**Unlock** ボタンをクリックして編集可能な状態にしてください
3. 標準ライブラリのデータセットはロックされていることがあります。その場合は **Duplicate** で複製してから編集してください

:::caution
ロックされたデータセットは意図的に保護されている場合があります。Unlock する前に、そのデータセットが他のシミュレーション設定から参照されていないか確認してください。
:::
