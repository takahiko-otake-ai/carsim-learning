# 用語集

CarSim に関連する主要な用語をアルファベット順にまとめました。

---

## A

**ADAS（Advanced Driver Assistance Systems）**
先進運転支援システムの総称。CarSim では ADAS 機能のシミュレーションが可能で、センサーモデルや Moving Object との連携により、ACC や AEB などの評価を行えます。

**Ackermann（アッカーマン）**
旋回時に内輪と外輪の転舵角差を幾何学的に理想化した操舵機構。CarSim ではアッカーマン率（%）としてパラメータ設定が可能です。

**Aerodynamics（空力）**
車両に作用する空気力学的な力とモーメント。CarSim では前面投影面積と 6 自由度の空力係数（Cd、Cl、Cy など）を用いて定義します。

**Animation（アニメーション）**
VS Visualizer で表示される 3D シミュレーション結果の視覚表現。車両挙動をリアルタイムまたは再生モードで確認できます。

## B

**Bushing（ブッシュ）**
サスペンションのリンク結合部に使用されるゴム製の弾性体。CarSim ではブッシュのコンプライアンス（変形量/力）としてモデル化されます。

**Brake Torque（ブレーキトルク）**
ブレーキシステムが車輪に加える制動トルク。マスターシリンダ圧力とブレーキトルク係数から算出されます。

## C

**Camber（キャンバー）**
車輪の傾き角度（車両正面から見た場合の垂直軸からの傾き）。サスペンションストロークに伴うキャンバー変化はキネマティクステーブルで定義します。

**CarSim**
Mechanical Simulation 社が開発した車両運動シミュレーションソフトウェア。乗用車やSUV の動的挙動を高精度に予測できます。

**Compliance（コンプライアンス）**
力やモーメントに対する変形量。サスペンションブッシュやステアリング系の弾性変形を表すために使用されます。

**Consolidated Parsfile (.cpar)**
CarSim のデータセットをエクスポートする際の統合ファイル形式。選択した Dataset と参照される全ての関連データを 1 つのファイルにまとめます。

**Copy and Link（コピー＆リンク）**
データセットを複製する操作の 1 つ。子データセットへのリンクを維持した状態でコピーを作成します。リンク先のデータは共有されます。

## D

**Database（データベース）**
CarSim のすべてのデータセットを格納するフォルダ構造。Database Builder で作成・管理します。

**Dataset（データセット）**
CarSim の GUI 上で 1 つの設定画面に対応するデータの単位。車両、タイヤ、道路など各カテゴリに個別のデータセットがあります。

**Differential（デファレンシャル）**
左右輪のトルク配分を制御する機構。オープン、LSD（リミテッドスリップ）、ロックなどのタイプを選択できます。

**Dongle（ドングル）**
USB 接続のハードウェアライセンスキー。CarSim のライセンス認証に使用されます。

**Driver Model（ドライバーモデル）**
仮想ドライバーの操舵・加減速を自動制御するモデル。速度追従やパスプレビューなどの制御ロジックを持ちます。

**Duplicate（デュプリケート）**
データセットを完全に独立したコピーとして複製する操作。元のデータセットとの関連は解除されます。

## E

**Echo File（エコーファイル）**
シミュレーション実行時に生成されるログファイル。パラメータの読み込み結果やエラーメッセージが記録され、デバッグに不可欠です。

**Events（イベント）**
シミュレーション中に特定の条件で動作を変更するトリガー。時間、距離、速度などの条件に基づいてパラメータを切り替えられます。

**Expanded Parsfile（拡張パースファイル）**
すべての参照ファイルを展開して 1 つのテキストファイルにまとめた Parsfile。デバッグやパラメータ確認に使用します。

**External Database（外部データベース）**
メインデータベースとは別に参照可能なデータベース。Library Tool を使ってデータセットをインポート・エクスポートできます。

## F

**FMI/FMU（Functional Mock-up Interface / Unit）**
シミュレーションツール間のモデル交換のための標準規格。CarSim は FMU としてエクスポート可能で、他のシミュレーション環境に組み込めます。

**Friction Circle（摩擦円）**
タイヤが発生できる縦力と横力の合力の限界を円（楕円）で表現した概念。タイヤの性能限界を理解するために使用されます。

## G

**GUI / VS Browser（グラフィカルユーザーインターフェース）**
CarSim のメイン操作画面。パラメータの設定、シミュレーションの実行、結果の確認をすべて GUI 上で行えます。VS Browser とも呼ばれます。

## H

**HIL（Hardware-in-the-Loop）**
実際の ECU ハードウェアをシミュレーション環境に組み込んで試験する手法。CarSim はリアルタイムソルバーを通じて HIL テストに対応しています。

**HEV（Hybrid Electric Vehicle）**
ハイブリッド電気自動車。CarSim ではエンジンとモーターを組み合わせたパワートレインモデルをシミュレーションできます。

## I

**Import/Export Channels（インポート/エクスポートチャネル）**
外部モデル（Simulink など）と CarSim ソルバー間でデータを受け渡すための信号定義。I/O チャネルとも呼ばれます。

## J

**Jounce（ジャウンス）**
サスペンションの圧縮方向のストローク（バンプとも呼ばれます）。反対方向のストロークはリバウンドと呼ばれます。

## K

**Kingpin（キングピン）**
操舵軸の傾斜角度を定義するパラメータ。キングピン傾角とキャスター角でステアリング軸の空間的な向きが決まります。

**Kinematics（キネマティクス）**
サスペンションのストロークに伴うホイールの幾何学的な動き（キャンバー変化、トー変化など）。テーブル形式で入力します。

## L

**Library（ライブラリ）**
CarSim のデータセットを整理・共有するための仕組み。標準ライブラリには参考データが多数収録されています。

**Library Tool（ライブラリツール）**
データセットのインポート・エクスポートを行うためのユーティリティ。異なるデータベース間でのデータ移動に使用します。

**LSD（Limited Slip Differential）**
リミテッドスリップデファレンシャル。トルクバイアス比やプリロードなどのパラメータで特性を定義します。

## M

**MacAdam Driver Model（マクアダムドライバーモデル）**
CarSim で使用される操舵制御のドライバーモデル。パスプレビューに基づく最適制御理論をベースとしています。

**Motion Sensor（モーションセンサー）**
車両上の任意の位置に配置できる仮想センサー。その位置での加速度、速度、変位などを出力できます。

**Moving Object（ムービングオブジェクト）**
シミュレーション空間内の他の車両や歩行者などの動的オブジェクト。ADAS シナリオの構築に使用されます。

## O

**OPT_SC**
CarSim の VS Commands で使用されるオプションキーワード。特定の制御ロジックやパラメータの動的変更を可能にします。

**Overlay（オーバーレイ）**
複数のシミュレーション結果を同じグラフ上に重ねて表示する機能。パラメータ比較やチューニングに非常に便利です。

## P

**Parsfile（パースファイル）**
CarSim のパラメータを記述するテキストファイル形式。人間が読める形式でパラメータが記述されており、テキストエディタで編集することも可能です。

**Path（パス）**
ドライバーモデルが追従する目標走行経路。S-L 座標系で定義されます。

**Path Preview（パスプレビュー）**
ドライバーモデルが前方の経路情報を先読みして操舵判断を行う機能。プレビュー距離とプレビュー時間で調整します。

**Procedure（プロシージャ）**
シミュレーションの実行手順を定義するデータセット。操舵入力、速度制御、道路条件などを組み合わせた試験シナリオを構成します。

**Powertrain（パワートレイン）**
エンジン/モーター、トランスミッション、デファレンシャルなど、車両の駆動系全体を指す総称です。

## R

**Road ID（ロードID）**
道路データセットを識別するための固有番号。シミュレーション内で複数の道路を参照する際に使用します。

**Roll Stiffness（ロール剛性）**
車両のロール運動に対する抵抗力。サスペンションばねとスタビライザーの合計で決まります。

**Run Control（ランコントロール）**
シミュレーションの実行条件を設定する画面。タイムステップ、終了条件、ソルバーパスなどを指定します。

**Run_all.par**
シミュレーション実行時にソルバーが最初に読み込むマスターパースファイル。すべての設定ファイルへの参照が含まれます。

## S

**S-Function（S-ファンクション）**
MATLAB/Simulink で CarSim ソルバーを呼び出すためのインターフェースブロック。Simulink モデル内に配置して共同シミュレーションを行います。

**S-L Coordinate（S-L座標系）**
道路中心線に沿った座標系。S は道路に沿った距離、L は道路中心線からの横方向オフセットを表します。

**Segment Builder（セグメントビルダー）**
道路のセグメント（直線、曲線など）を組み合わせて走行コースを作成するツールです。

**Simfile（シムファイル）**
CarSim が Simulink 連携時に使用する設定ファイル。ソルバーの初期化に必要な情報が含まれます。

**Similarity Method（類似法）**
既存のタイヤデータを基にスケーリングすることで、異なるサイズのタイヤ特性を推定する手法です。

**Solver（ソルバー）**
車両の運動方程式を数値的に解くプログラム本体。CarSim の計算エンジンに相当します。

**Sprung Mass（ばね上質量）**
サスペンションのばねで支持される車体部分の質量。車両総質量からばね下質量を差し引いた値です。

## T

**Tire Model（タイヤモデル）**
タイヤの力発生特性を数学的に表現するモデル。CarSim では内蔵テーブルモデルのほか、外部タイヤモデル（MF-Tire など）も使用できます。

**Torque Vectoring（トルクベクタリング）**
左右輪や前後輪間のトルク配分を能動的に制御する技術。CarSim では VS Commands や外部コントローラーで実装できます。

**Turn Slip（ターンスリップ）**
低速旋回時にタイヤの接地面内で発生する回転方向のすべり。駐車時のステアリングトルク予測に影響します。

## U

**Unsprung Mass（ばね下質量）**
サスペンションのばねより下側の部品（ホイール、ブレーキ、アクスルなど）の質量です。

## V

**Vehicle Code（ビークルコード）**
CarSim の製品バリエーションを識別するコード（例: i_i）。ライセンスに含まれる Vehicle Code によって使用可能な機能が決まります。

**VS API（VS アプリケーションプログラミングインターフェース）**
C/C++ プログラムから CarSim ソルバーを直接呼び出すためのプログラミングインターフェースです。

**VS Commands（VS コマンド）**
Parsfile 内でパラメータの動的変更や条件分岐を記述するためのテキストコマンド体系です。

**VS COM Interface（VS COM インターフェース）**
Windows の COM 技術を使用して CarSim の GUI を外部プログラムから制御するインターフェースです。

**VS Connect**
ネットワーク経由で複数の CarSim インスタンスや外部システムを接続するための通信フレームワークです。

**VS STI（VS Standard Tire Interface）**
外部タイヤモデルを CarSim に組み込むための標準インターフェース。MF-Tire や FTire などのサードパーティ製タイヤモデルに対応しています。

**VS Terrain（VS テレイン）**
3D の路面形状を定義する機能。高さマップや 3D メッシュを使った複雑な路面を表現できます。

**VS Visualizer（VS ビジュアライザー）**
CarSim のシミュレーション結果を 3D アニメーションで表示するビューアです。車両挙動の視覚的な確認に使用します。
