# よくある質問 (FAQ)

CarSim を使用する際によく寄せられる質問とその回答をまとめました。

---

## インストール・ライセンス

### Q: CarSim のインストール手順を教えてください

**A:** 以下の手順でインストールします。

1. [carsim.com](https://www.carsim.com) からインストーラをダウンロードします
2. インストーラを実行し、画面の指示に従ってインストールを完了します
3. CarSim を起動し、**Database Builder** を使って新しいデータベースを作成します

:::note
インストール先のフォルダパスには日本語や特殊文字を含めないでください。英数字のみのパスを推奨します。
:::

### Q: ドングルライセンスとネットワークライセンスの違いは何ですか？

**A:** CarSim では 2 種類のライセンス方式があります。

- **ドングルライセンス**: USB ドングルをPCに接続して使用します。ドングルを挿したPCでのみ CarSim を実行できます。オフライン環境でも使用可能です。
- **ネットワークライセンス**: ライセンスサーバーからネットワーク経由でライセンスを取得します。複数ユーザーでライセンスを共有する場合に適しています。ネットワーク接続が必要です。

ライセンスの設定は、CarSim GUI の **License Settings** 画面で行います。

### Q: 複数のデータベースを持つことはできますか？

**A:** はい、可能です。**Database Builder** を使用して、目的別に複数のデータベースを作成できます。例えば、プロジェクトごとにデータベースを分けて管理することができます。CarSim 起動時にどのデータベースを開くか選択できます。

## シミュレーション

### Q: シミュレーションが実行できません。どうすればよいですか？

**A:** 以下の項目を順番に確認してください。

1. **ライセンス**: ドングルが正しく接続されているか、またはネットワークライセンスサーバーに接続できているか確認します
2. **ログファイル**: Results フォルダ内のログファイル（Echo File）にエラーメッセージがないか確認します
3. **ソルバーパス**: Run Control 画面でソルバー（Solver）のパスが正しく設定されているか確認します
4. **Vehicle Code**: 使用している Vehicle Code がライセンスで許可されているか確認します

:::tip
Echo File はシミュレーション実行時に生成され、パラメータの読み込み状況やエラーの詳細が記録されます。問題解決の最も重要な手がかりになります。
:::

### Q: タイムステップを変更するにはどうすればよいですか？

**A:** **Run Control** 画面で `Time Step` パラメータを変更できます。デフォルト値は **0.0005 秒**（0.5 ms）で、ほとんどのケースでこの値が推奨されます。タイムステップを大きくすると計算は速くなりますが、数値安定性や精度に影響する場合があります。

### Q: 複数のシミュレーション結果を比較するにはどうすればよいですか？

**A:** **Overlay** 機能を使用します。

1. 比較したい Run を **Overlay** ウィンドウに追加します
2. **Set Color** ボタンで各 Run に異なる色を割り当てます
3. Plot 画面で複数の結果が重ね合わせて表示されます

Overlay は最大数十の Run を同時に比較でき、パラメータスタディに非常に有用です。

## データ管理

### Q: シミュレーション設定を他のユーザーと共有するにはどうすればよいですか？

**A:** **Consolidated Parsfile (.cpar)** としてエクスポートします。cpar ファイルには、選択した Dataset とそれが参照するすべての関連データが含まれます。受け取り側は CarSim の Import 機能で cpar ファイルを読み込むことで、同じ設定を再現できます。

### Q: 「Duplicate」と「Copy and Link」の違いは何ですか？

**A:** 両者はデータセットの複製方法が異なります。

| 操作 | 動作 | 元データとの関係 |
|---|---|---|
| **Duplicate** | 完全に独立したコピーを作成 | 元データと無関係（変更が互いに影響しない） |
| **Copy and Link** | リンク付きのコピーを作成 | 子データセットが共有される（片方の変更が他方にも反映される） |

用途に応じて使い分けてください。パラメータスタディでは Duplicate、共通設定を維持したい場合は Copy and Link が便利です。

### Q: データベースのバックアップ方法を教えてください

**A:** 2 つの方法があります。

1. **フォルダコピー**: データベースフォルダ全体をコピーします（最もシンプルな方法）
2. **Library Tool**: CarSim の Library Tool を使用して、必要なデータセットを選択的にエクスポートします

定期的なバックアップを推奨します。

## 外部ツール連携

### Q: どのバージョンの MATLAB/Simulink に対応していますか？

**A:** 対応する MATLAB/Simulink のバージョンは CarSim のバージョンによって異なります。各リリースの **Release Notes** で対応バージョンの一覧を確認してください。一般的に、CarSim リリース時点で最新の数バージョンの MATLAB がサポートされます。

### Q: Simulink なしで CarSim を使えますか？

**A:** はい、使用できます。CarSim はスタンドアロンモードで単独実行が可能です。さらに以下のインターフェースも利用できます。

- **VS Commands**: テキストベースのコマンドで外部入力を定義
- **Python API**: Python スクリプトから CarSim を制御
- **VS API**: C/C++ プログラムから直接ソルバーを呼び出し
- **FMI/FMU**: 標準インターフェースを使った他のシミュレーションツールとの連携

### Q: dSPACE SCALEXIO との接続方法を教えてください

**A:** CarSim GUI の **Help > Real-Time Systems > dSPACE RT Guide** に詳細なガイドが掲載されています。基本的な流れは以下の通りです。

1. CarSim で RT（リアルタイム）用のモデルをビルドします
2. dSPACE ConfigurationDesk でモデルを取り込みます
3. I/O 信号のマッピングを設定して実行します

:::caution
リアルタイムシミュレーションでは、タイムステップの設定がハードウェアの処理能力に見合っていることを確認してください。
:::
