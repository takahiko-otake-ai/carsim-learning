# 演習3: データのエクスポートとインポート

## 学習目標

- CarSim データセットの **エクスポート（書き出し）** 方法を習得する
- 新規データベースの作成方法を理解する
- エクスポートしたデータの **インポート（読み込み）** 手順を学ぶ
- Consolidated（.cpar）と Expanded（.par）のエクスポート形式の違いを理解する

## 前提条件

- [演習1](./exercise-1.md) を完了していること
- `** CS Training` カテゴリに `#1 C-Class, Acceleration` が作成済みであること

---

## エクスポート形式について

CarSim には2種類のエクスポート形式があります。

| 形式 | 拡張子 | 特徴 |
|---|---|---|
| **Consolidated** | `.cpar` | すべての関連データを1つのファイルにまとめる。共有やバックアップに最適 |
| **Expanded** | `.par` | 各データセットを個別のファイルとして書き出す。バージョン管理や差分比較に適する |

この演習では **Consolidated (.cpar)** 形式を使用します。

---

## 手順

### ステップ 1: エクスポート対象の選択

ツリーから **&#123;** CS Training&#125; #1 C-Class, Acceleration** に移動します。Home 画面にこのデータセットが表示されていることを確認してください。

### ステップ 2: エクスポートの実行

1. メニューバーから **File → Export Consolidated Parsfile...** を選択します
2. 保存先のパスを指定します

:::warning
保存先のフォルダパスには**英語（半角英数字）のみ**を使用してください。日本語や特殊文字を含むパスではエクスポートに失敗する場合があります。
:::

3. 確認ダイアログでチェックボックスをオンにし、**OK** をクリックします
4. エクスポートが完了するまで待ちます

### ステップ 3: 新規データベースの作成

1. **File → Open Database Builder** を選択します
2. Database Builder ウィンドウで **Minimal** テンプレートを選択します
3. **Build Database from Selected Items** をクリックしてデータベースを構築します

### ステップ 4: 新しいデータベースへの切り替え

1. **Switch (Same Window)** を選択して、新しく作成したデータベースに切り替えます
2. ライセンス設定画面が表示された場合は、演習1と同様に設定します

:::info
Minimal テンプレートで作成されたデータベースには、最小限のデータセットのみが含まれます。ここにエクスポートしたデータをインポートすることで、データの移行を確認できます。
:::

### ステップ 5: データのインポート

1. **File → Import Parsfile (Any Export Type)...** を選択します
2. ステップ 2 で保存した `.cpar` ファイルを選択します
3. **Import Settings** ダイアログが表示されたら、設定を確認して **OK** をクリックします
4. **Preview** ダイアログでインポートされるデータセットの一覧を確認し、**OK** をクリックします

### ステップ 6: インポート結果の確認

インポートが完了したら、以下の点を確認します:

- `** CS Training` カテゴリが表示されていること
- `#1 C-Class, Acceleration` データセットが正しくインポートされていること
- リンクされている Vehicle、Procedure などのサブデータセットも含まれていること
- シミュレーションを実行して、元のデータベースと同じ結果が得られること

:::tip
インポート後にシミュレーションを実行し、結果がエクスポート前と一致することを確認するのがベストプラクティスです。これにより、すべてのデータが正しく移行されたことを検証できます。
:::

---

## まとめ

エクスポートとインポートの機能は、以下の場面で活用できます:

- **チーム間でのデータ共有**: Consolidated 形式で1ファイルにまとめて送付
- **バックアップ**: 重要なデータセットの保存
- **異なるバージョンのデータベースへの移行**: 新バージョンの CarSim へのデータ移行
- **トラブルシューティング**: サポートチームへのデータ提供
