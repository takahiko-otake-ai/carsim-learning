# 演習5: Simulink 連携

## 学習目標

- CarSim と MATLAB/Simulink の連携方法を理解する
- **S-Function** の概念と役割を学ぶ
- Simulink モデルへのインポート/エクスポートチャネルの設定方法を習得する
- 簡単なブレーキ制御モデルを構築し、連携シミュレーションを実行する

## 前提条件

- [演習2](./exercise-2.md) を完了し、`#2 C-Class, 60 km/h` が作成済みであること
- **MATLAB/Simulink R2022b** 以降がインストール済みであること
- MATLAB と Simulink の基本操作に慣れていること

:::warning
MATLAB/Simulink のバージョンによっては互換性の問題が発生する場合があります。CarSim のリリースノートで対応バージョンを確認してください。
:::

---

## S-Function とは

CarSim の **S-Function（System Function）** は、Simulink 環境内で CarSim の車両モデルをブロックとして使用するためのインターフェースです。Simulink モデルから CarSim にコントロール信号（ブレーキ圧、ステアリング角など）を送り、CarSim からの車両状態（速度、加速度など）を受け取ることができます。

```
┌─────────────┐     Import      ┌──────────┐     Export      ┌─────────────┐
│  Simulink   │ ──────────────→ │  CarSim  │ ──────────────→ │  Simulink   │
│  コントローラ │   (制御入力)     │ S-Function│   (車両状態)     │  結果処理   │
└─────────────┘                 └──────────┘                 └─────────────┘
```

## 手順

### ステップ 1: データセットの準備

1. `#2 C-Class, 60 km/h` に移動します
2. **Duplicate** を実行します
   - **Category**: `** CS Training`
   - **Title**: `#5 C-Class, Braking from Simulink`

### ステップ 2: Simulink モデルの選択

**Models** タブを開き、**Models: Simulink** を選択します。これにより、CarSim のソルバーが単体実行モードから Simulink 連携モードに切り替わります。

### ステップ 3: ABS コントローラの設定

1. ABS Controller データセットをリンクします
2. リンクしたデータセットを右クリックし、**Copy and Link** を選択します
3. 新しい名前: `M/C Input`

### ステップ 4: Simulink モデルファイルの配置

CarSim インストールフォルダ内の `Extensions\Simulink` から `training_mc_input.mdl` をコピーし、作業フォルダに配置します。

Simulink モデルファイルのパスを設定画面で指定します。

:::info
`.mdl` ファイルが見つからない場合は、CarSim のインストールディレクトリ（通常は `C:\Program Files\CarSim_20XX.X\`）の `Extensions\Simulink` フォルダを確認してください。
:::

### ステップ 5: インポートチャネルの設定

インポートチャネルは、Simulink から CarSim に送信する制御信号を定義します。

1. 既存のインポートチャネルリストを**クリア**します
2. **Select by component type** から **Brakes** を選択します
3. チャネルリストから **IMP_PCON_BK**（ブレーキマスターシリンダー圧力）を追加します

:::tip
`IMP_PCON_BK` は、マスターシリンダーのブレーキ圧力入力を表します。Simulink 側からこのチャネルに圧力値を送ることで、ブレーキ制御が可能になります。
:::

### ステップ 6: エクスポートチャネルの設定

この演習ではエクスポートチャネルは使用しないため、既存のエクスポートチャネルを**すべて削除**します。

### ステップ 7: シミュレーション時間の設定

**Stop time** を **10 秒** に設定します。

### ステップ 8: プロットの設定

以下のプロットをリンクして、結果の確認に使用します:

- **Brake Control: M/C Pressure** — ブレーキ圧力の時系列
- **Longitudinal Speed** — 車両の前後方向速度

### ステップ 9: Simulink モデルの構築

**Send to Simulink** をクリックして Simulink を起動します。以下のブロック図を構築します:

```
                    ┌──────────┐
  ┌───────┐        │          │        ┌──────────────┐
  │ Clock │──────→ │  Switch  │──────→ │   CarSim     │
  └───────┘        │(threshold│        │  S-Function  │
                   │   = 3)   │        └──────────────┘
  ┌───────────┐    │          │
  │Constant:10│──→ │          │
  └───────────┘    │          │
  ┌───────────┐    │          │
  │Constant: 0│──→ │          │
  └───────────┘    └──────────┘
```

モデルの動作:

- **Clock**: シミュレーション時間を出力
- **Switch**: 時間が 3 秒を超えたら Constant 10（ブレーキ圧力 10 MPa）を、それ以前は Constant 0（ブレーキなし）を出力
- **CarSim S-Function**: ブレーキ圧力入力を受けて車両モデルを計算

:::caution
Switch ブロックの **Threshold** パラメータを正確に **3** に設定してください。この値により、シミュレーション開始後 3 秒間は自由走行、その後ブレーキが作動する動作になります。
:::

### ステップ 10: シミュレーションの実行と結果確認

1. CarSim の画面で **Simfile** の名前が正しく設定されていることを確認します
2. Simulink の **Run** ボタンをクリックしてシミュレーションを実行します
3. CarSim 側で **Video + Plot** を開き、結果を確認します

期待される結果:

- 最初の 3 秒間: 車両は 60 km/h で定速走行
- 3 秒以降: ブレーキ圧力が 10 MPa で印加され、車両が減速
- プロットでブレーキ圧力のステップ入力と速度の低下が確認できる

---

## まとめ

この演習で、CarSim と Simulink の連携の基本的なワークフローを学びました。S-Function を使用することで、Simulink 上で設計した制御ロジック（ABS、ESC、ADAS など）を CarSim の高精度な車両モデルと組み合わせてテストできます。

:::tip
応用として、以下のような拡張を試してみましょう:
- ブレーキ圧力を PID コントローラで制御し、目標減速度を実現する
- 車輪速度のエクスポートチャネルを追加し、ABS ロジックを実装する
- ステアリング入力も Simulink から制御する
:::
